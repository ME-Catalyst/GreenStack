# GreenStack Documentation

This folder now groups documentation into logical collections so information is easier to find and maintain. Use this index to jump to the most relevant references.

## Guides
- `docs/guides/DEVELOPER_GUIDE.md` – Full-stack development workflow, tooling, and standards.
- `docs/guides/CONTRIBUTING.md` – Contribution process (branching, reviews, testing requirements).
- `docs/guides/TROUBLESHOOTING.md` – Common investigation playbooks and log locations.
- `docs/guides/ACCESSIBILITY.md` – Frontend accessibility requirements and validations.
- **Operations**
  - `docs/guides/operations/DEPLOYMENT_RUNBOOK.md` – Deployment procedures for Windows/Linux along with Docker steps.
  - `docs/guides/operations/DISASTER_RECOVERY.md` – Backup, restore, and incident recovery execution plans.
  - `docs/guides/operations/MONITORING_SETUP_GUIDE.md` – Observability stack, alerting, and health checks.

## References
- `docs/references/ARCHITECTURE.md` – System architecture (API, services, diagrams, deployment topology).
- `docs/references/DATABASE_SCHEMA.md` – Entity-relationship diagrams and Alembic migration history.
- `docs/references/COMPONENTS_INVENTORY.md` – Frontend component catalog with usage guidance.
- `docs/references/PQA_ARCHITECTURE.md` – Parser Quality Assurance design and workflows.
- **API**

  - `docs/references/api/API_REFERENCE.md`

## Reports & Summaries
- `docs/reports/CHANGELOG.md` – Release notes and feature history.
- `docs/reports/PQA_IMPLEMENTATION_COMPLETE.md` – PQA delivery summary.
- `docs/reports/WEEK_8_FINAL_SUMMARY.md` – Weekly program recap.
- Extraction, refactoring, and analysis reports live under `docs/reports/{extraction|refactoring|analysis}/`.

## Audits
`docs/audits/` hosts the complete multi-phase audit program (security, database, CI/CD, accessibility, etc.) plus the master action plan.

## Archive
Historical documents now live under `docs/archive/` (v1 architecture, early theming plans, sprint summaries, and prior deployment notes). See `docs/archive/README.md` for catalog details.
